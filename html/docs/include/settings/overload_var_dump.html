Xdebug can replace PHP's bulit-in var_dump() function with its own implementation. The extra information includes data types, size of arrays, and object IDs. The overloaded function respects the [CFG:var_display_max_children], [CFG:var_display_max_data], and [CFG:var_display_max_depth] settings.

In an HTML environment, when PHP's html_errors set to 1, Xdebug's version will use HTML tags to format the information.

In a CLI environment, Xdebug's version can also include colours, depending on the value of [CFG:cli_color].

This setting can have the following values:

0

Xdebug will not change PHP's built-in var_dump() function to an improved implementation.

1

Xdebug replaces PHP's built-in var_dump() function with an implementation that shows the described extra information.

2 (the default)

The overloaded var_dump() output will also include the filename and line number to the output. The [CFG:file_link_format] setting is respected for the formatting of this information.
